# Contributing to Retro Handheld Console

Thank you for your interest in contributing to the Retro Handheld Console! We welcome contributions to improve the engine, add new games, or enhance the documentation.

## 🚀 How to Contribute

### Reporting Bugs
If you find a bug, please check the [Issues](https://github.com/s0um1kx/retro-handheld-console/issues) page to see if it has already been reported. If not, feel free to open a new issue with:
- A clear description of the problem.
- Steps to reproduce the bug.
- Browser information (if applicable).

### Adding New Games
We love new game contributions! To add a game to the `src/games/` folder:
1. Fork this repository.
2. Create a new folder for your game under `src/games/`.
3. Ensure your game uses the core engine logic found in `src/core/`.
4. Submit a Pull Request (PR) with a brief description of your game.

### Code Style
To keep the codebase clean and professional:
- Use **2-space indentation**.
- Follow existing naming conventions (e.g., `PascalCase` for classes).
- Ensure your code is well-commented, especially for complex game logic.

## 🤝 Pull Request Process
1. Update the documentation if you add new features.
2. Ensure your changes are modular and do not break the core engine.
3. Once your PR is submitted, it will be reviewed for code quality and functionality.
